# 技术文章审阅技能

帮助用户审阅和评估中文技术博客文章的质量，确保文章符合高质量技术写作标准。

## 审阅维度

按以下六个维度评估文章质量。每个维度的详细评估标准见 `references/review-dimensions.md`：

1. **问题驱动**（Why > What）— 文章是否以清晰的技术问题开始
2. **技术脉络**（Context）— 是否交代了技术出现的背景和时空环境
3. **方案对比**（How > What）— 是否提供了多种解决方案的比较
4. **推导过程**（Problem → Model → Solution）— 是否有清晰的推导逻辑
5. **原理与方法论**（Know-Why）— 是否触及原理层面，让读者一通百通
6. **资料来源**（References）— 是否引用了权威资料

## 审阅流程

1. **读取文章**。如果用户提供了文件路径，先读取文章内容。
2. **逐维度评估**。按照上述六个维度逐一评估，参考 `references/review-dimensions.md` 中的具体标准。
3. **按需增强严谨度**。如果用户要求严苛审阅、证据审计、同行评审或尽调式检查，在本技能内增加两项检查：证据强度是否支撑原文结论、是否存在方法缺陷、逻辑跳跃、过度声称或无支撑断言。
4. **给出评分**。每个维度给出 1-5 分评分（1=很差，3=合格，5=优秀）。
5. **指出问题**。具体指出文章在每个维度上的问题。
6. **提供建议**。给出可操作的改进建议。
7. **总结**。给出总体评价和优先改进项。

输出审阅报告时，使用 `assets/report-template.md` 中的模板格式。

## 审阅原则

- **建设性**：指出问题的同时给出具体改进建议
- **客观性**：基于标准评估，不掺杂个人偏好
- **优先级**：帮助用户识别最重要的改进项
- **可操作性**：建议要具体，用户知道怎么改

## 注意事项

- 如果文章是英文的，仍然使用这些标准进行审阅，但输出报告使用中文
- 如果用户只提供了文章片段，基于片段进行评估，并说明无法评估的部分
- 不要修改用户的文章，只提供审阅意见
- 如果文章质量已经很高，也要指出可以进一步提升的地方

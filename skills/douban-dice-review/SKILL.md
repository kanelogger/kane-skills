# 骰子豆瓣影评

用六个固定骰子为一部电影抽取六个理论关键词，再把六个关键词各写成一句话，组成一份豆瓣风格中文影评。

## 轻搜策略

这个技能首先是娱乐写作，不是严肃影评研究。默认不要搜索，保持骰子随机和即时输出的节奏。

只在这些情况做一次 30 秒轻搜：

- 用户只给片名，没有剧情、人物、观感或具体材料；
- 用户要求“结合电影内容”“写得像看过”“带一点具体细节”；
- 片名明显可能同名，直接写会有较高错片风险；
- 电影很冷门、很新，或你对基本信息没有把握。

这些情况不要搜索：

- 用户已经给出剧情、人物、场景、观后感或指定素材；
- 用户说“随便写”“抽象一点”“别查资料”“纯骰子”；
- 用户要的是荒诞理论腔短评，不要求贴合具体剧情。

轻搜只拿电影事实素材，不拿别人观点。目标限于：

```text
片名 / 年份 / 导演
类型 / 国家地区
一句话剧情
主要人物关系
2-3 个关键场景、意象或设定
明显主题词
```

优先使用这些查询：

```text
《电影名》 年份 导演 剧情 简介
《电影名》 主要人物 情节
《电影名》 视觉风格 主题
Movie Title film year director plot
Movie Title synopsis cast themes
Movie Title key scenes visual style
```

同名歧义时使用：

```text
电影名 site:wikipedia.org film
电影名 IMDb year director
电影名 豆瓣 电影 年份 导演
```

不要搜索或搬运这些内容：

```text
豆瓣短评
专业影评
解析文章
结局解读
“隐喻是什么”类内容
```

如果执行了轻搜，最终输出在本轮汇总前加一行 `轻搜素材：...`，只列 2-4 个可用于影评的事实物件。资料不确定时，用“如果按公开梗概理解”这类限定，不要把猜测写成电影事实。

## 固定顺序

轻搜判断完成后，每次都按这个顺序执行，不要跳步、合并或改写骰子表。

### 1. 准备六个骰子

```text
1. 理论家：1 弗洛伊德，2 齐泽克，3 拉康，4 福柯，5 本雅明，6 德勒兹
2. 电影讨论的是啥：1 欲望，2 创伤，3 记忆，4 身份，5 权力，6 异化
3. 分析对象：1 身体，2 家庭，3 空间，4 时间，5 影像，6 语言
4. 高级名词：1 他者，2 凝视，3 缺席，4 能指，5 规训，6 主体
5. 格局打开：1 现代性，2 后现代性，3 技术理性，4 资本主义，5 数字时代，6 景观社会
6. 给出结论：1 主体性的瓦解，2 身份认同危机，3 归属感消失，4 现实边界模糊，5 日常经验陌异化，6 意义系统崩塌
```

### 2. 用脚本获取随机结果

运行技能目录中的脚本：

```bash
node [skill-dir]/scripts/roll_dice.js
```

如果当前环境没有 `node`，用 `bun` 运行同一个脚本：

```bash
bun [skill-dir]/scripts/roll_dice.js
```

脚本必须用下面两个函数完成随机抽取：先对当前骰子的 6 个面乱序，再取乱序数组最后一项。

```js
// 数组乱序
function shuffle(arr) {
  let i = arr.length;
  while (i) {
    let j = Math.floor(Math.random() * i--);
    [arr[i], arr[j]] = [arr[j], arr[i]]
  }
  return arr
}

// 获取数组中的最后一项
function getLstMember(num = 1) {
  return array.slice(-num)
}
```

### 3. 输出本轮汇总

脚本会输出六个骰子的点数和关键词。把结果整理成下面格式并先呈现给用户：

```markdown
本轮汇总：理论家=[关键词] / 议题=[关键词] / 分析对象=[关键词] / 高级名词=[关键词] / 时代框架=[关键词] / 结论=[关键词]
```

### 4. 根据关键词写一份影评

根据六个关键词输出一份影评。规则：

- 每个关键词只写一句话，最终正文固定 6 句。
- 六句话顺序必须对应六个骰子：理论家、议题、分析对象、高级名词、时代框架、结论。
- 每句话必须显式包含对应关键词，并让六句话连成一份完整影评。
- 影评正文不要带序号、项目符号、骰子标签或小标题；也不要写成 `1.`、`-`、`理论家：` 这类清单。
- 写成中文豆瓣影评口吻：有判断、有电影感、有一点理论腔，但不要写成论文提纲。
- 基于搜索到的电影信息或用户给出的素材写具体细节；没有可靠信息时，避免编造剧情事实，用更抽象的观影描述完成。
- 最后一句必须落到第 6 个骰子的结论词。

## 输出格式

```markdown
[轻搜素材：片名/年份/导演/关键场景或设定；未执行轻搜时省略本行]

本轮汇总：理论家=[关键词] / 议题=[关键词] / 分析对象=[关键词] / 高级名词=[关键词] / 时代框架=[关键词] / 结论=[关键词]

影评：
[包含理论家关键词的一句话。]
[包含议题关键词的一句话。]
[包含分析对象关键词的一句话。]
[包含高级名词关键词的一句话。]
[包含时代框架关键词的一句话。]
[包含结论关键词的一句话。]
```

## 失败处理

如果脚本无法运行，手动按同一骰子表执行一次随机抽取，并明确说明“脚本未运行，已手动按同一规则抽取”。仍然要输出本轮汇总和六句影评。

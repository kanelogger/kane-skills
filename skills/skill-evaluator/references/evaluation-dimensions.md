# 五大测评维度详解

## 1. 功能正确性（Functional Correctness）— P0

> 回答："这次任务到底做成了没？"

| 子维度 | 定义 | 评测方法 | 典型指标 |
|---|---|---|---|
| 结果正确性 | 最终产出是否符合预期 | 代码比对、单元测试、数据库校验 | pass@1 / pass^k |
| 任务完成度 | 多步任务完成的百分比 | 子目标打点 | 完成率 % |
| 指令遵循度 | 是否严格按用户指令输出 | JSON Schema 校验、正则、字段检查 | 遵循率 % |
| 工具调用正确性 | 是否选对工具、参数是否正确 | 调用日志断言 | 调用准确率 % |

**必须自动化、全覆盖**。这是确定性评分器的主战场。

## 2. 过程质量（Process Quality）— P1

> 回答："即便做对了，过程合理吗？"

| 子维度 | 定义 | 评测方法 | 典型指标 |
|---|---|---|---|
| 推理合理性 | 思考链条是否自洽、没有跳步 | Rubric 评委 | 合理性评分 1-5 |
| 步骤最优性 | 是否走了不必要的弯路 | 比较实际步数 vs 参考解法 | 步数比 |
| 信息完整性 | 输出是否覆盖了所有关键信息 | Rubric 按要点核查 | 要点覆盖率 % |
| 上下文利用率 | 是否充分利用了提供的上下文 | Rubric + 人工抽查 | 利用率评分 |
| 自我纠错能力 | 遇到错误时是否能识别并修正 | Trace 分析 | 纠错成功率 |

**Rubric 评委的主战场**，最能体现"智能"水平。

## 3. 效率与成本（Efficiency & Cost）— P1

> 回答："结果对了，但划得来吗？"

| 子维度 | 定义 | 评测方法 | 典型指标 |
|---|---|---|---|
| Token 消耗 | 输入 + 输出 token 总量 | API 统计 | avg / p95 tokens |
| 工具调用次数 | 完成任务的工具调用总数 | Trace 统计 | avg / p95 calls |
| 端到端延迟 | 用户发起到收到结果的时间 | 时间戳 | p50 / p95 / p99 latency |
| 失败重试率 | 单次试验内的重试次数 | Trace 统计 | 重试率 % |
| 单次任务成本 | 折算成人民币/美元 | Token × 单价 | ¥/task |

**每个任务都应带上成本画像**。一个 pass@1 高但 Token 花 10 倍的方案在生产上不可接受。

## 4. 鲁棒性与安全（Robustness & Safety）— P0

> 回答："关键时刻翻车吗？"

| 子维度 | 定义 | 评测方法 | 典型指标 |
|---|---|---|---|
| 一致性/稳定性 | 相同输入多次运行结果是否一致 | 多次试验（k=5/10）| pass^k |
| 异常恢复 | 工具失败、超时、返回异常时能否兜底 | 故障注入测试 | 恢复成功率 |
| 抗对抗/抗注入 | 面对 Prompt Injection 是否守住 | 红队用例集 | 抗攻击率 |
| 幻觉率 | 是否编造不存在的事实/API/字段 | 事实核查 | 幻觉率 % |
| 越权/越界风险 | 是否执行了超出授权的操作 | 权限断言 | 越权次数 |
| 合规性 | 是否泄露 PII、违反行业规范 | 正则 + 人工抽查 | 违规率 |
| 拒绝合理性 | 该拒是否拒、不该拒是否过度拒 | Rubric + 人工 | 误拒%/漏拒% |

**涉及金融、医疗、企业数据的 Agent 必须前置此类测评**。

### pass^k vs pass@k

- **pass@k**：k 次试验中**至少 1 次**通过的概率 → 衡量**峰值能力**
- **pass^k**：k 次试验中**每次都**通过的概率 → 衡量**稳定性**

## 5. 体验与对齐（Experience & Alignment）— P2

> 回答："用户愿意继续用吗？"

| 子维度 | 定义 | 评测方法 | 典型指标 |
|---|---|---|---|
| 语气风格 | 是否符合品牌调性 | Rubric 评委 | 风格评分 |
| 回复清晰度 | 结构是否清楚、有无废话 | Rubric + 人工 | 清晰度评分 |
| 主动澄清 | 模糊需求时是否主动提问 | Rubric | 澄清率 % |
| 同理心 | 是否能识别情绪并恰当回应 | 人工 + Rubric | 同理心评分 |
| 可解释性 | 是否能说清自己做了什么 | 人工抽查 | 可解释评分 |
| 用户满意度 | 真实用户反馈 | 线上 👍/👎、NPS | CSAT / NPS |

早期用 Rubric + 人工抽样，成熟后引入线上 A/B + 用户反馈闭环。

## 不同 Skill 类型的测评侧重

| Skill 类型 | 核心风险 | 测评侧重 | 主要评委 |
|---|---|---|---|
| 知识库问答 | 幻觉、引用错误 | 准确性、引用溯源 | Rubric |
| 代码编写 | 无法编译/运行 | 产物正确性、可运行性 | 确定性 |
| 功能工具 | 工具调用错误 | 过程合规性、工具调用正确性 | 确定性 + 过程对比 |
| 问题定位 | 根因误判 | 推理链路、根因准确性 | 过程 + 结果双重验证 |
